# 제1절 전자조달시스템 개요

## 1.1 전자조달시스템 정의

**전자조달(e-Procurement) 시스템**: 인터넷 등 정보통신 네트워크를 기반으로 상품·용역의 소싱, 구매, 지불 등을 자동화로 처리할 수 있는 전자적 처리 프로세스의 집합

### 주요 정의
| 기관 | 정의 |
|------|------|
| **세계은행(2003)** | 정부가 공공부문에서 요구하는 공사·상품·서비스를 조달하기 위해 공급업체와 조달관계를 수행하는 데 정보기술(특히 인터넷)을 사용하는 것 |
| **EU** | 공공조달 절차 전반에 걸친 ICT 기반 업무처리 시스템 |

### 전자조달시스템이 구현하는 절차
입찰공고 → 입찰문서 제공 → 입찰서 제출 → 평가 → 수주 → 주문 → 송장 발행 → 지불

## 1.2 전자조달시스템의 유형

영국 OGC(2005, 현 CCS)는 전자조달시스템을 **전자적 소싱(e-Sourcing)** + **전자적 구매(e-Purchasing)** 2가지 유형으로 범주화했다.

![전자조달시스템 2가지 유형 (영국 OGC 분류)](/procurement-wiki/images/textbook/book1/p131_fig1.png)

> **학습 포인트**: 전자조달시스템을 소싱(공급자 탐색·평가)과 구매(주문·지불 처리) 2가지 유형으로 분류한 영국 OGC 2005년 분류체계이다. 시험에서 "e-Sourcing은 입찰·평가·협상 중심, e-Purchasing은 카탈로그·지불·송장 중심"이라는 구분을 가장 자주 묻는다.

### 가. 전자적 소싱(e-Sourcing) 시스템
공급자를 찾고 평가하는 과정 지원에 초점

| 기능 | 설명 |
|------|------|
| e-Tendering | 전자입찰 |
| e-Evaluation | 입찰업체 제안서 온라인 평가 |
| e-Collaboration | 공동구매를 위한 전자적 협업 |
| e-Negotiation | 전자적 협상 |
| e-Auction | 전자적 경매 |
| Contract Management | 계약관리 |

### 나. 전자적 구매(e-Purchasing) 시스템
정확한 주문정보 기반의 구매·지불 처리에 초점

| 기능 | 설명 |
|------|------|
| e-Purchasing | 전자적 구매 처리 |
| e-Payment | 전자적 지불 |
| Purchasing Card | 정부구매카드(P-card) |
| e-Invoicing | 전자송장 |

## 1.3 전자조달시스템 운영 환경 (지출분석 4사분면)

지출 규모(Spend)와 시장 난이도(Difficulty = 복잡성 + 위험)의 2차원으로 분석한다.

![전자조달시스템 구축 유형 및 기능 결정 사분면 (Schapper, 2006)](/procurement-wiki/images/textbook/book1/p132_fig1.png)

> **학습 포인트**: 지출규모 × 시장 난이도(복잡성+위험) 2차원으로 공공조달 지출을 4사분면(기술적/전략적/인수/레버리지)으로 분류한 Schapper(2006) 모델이다. 시험에서는 "전략적 지출 = 공공조달에서 비중이 가장 높음 → e-Tendering 중심"이 가장 중요되며, 4사분면별 중점 기능(계약관리/e-Tendering/e-Catalogue/e-Auction)을 묻는 문제가 핵심된다.

### 4가지 지출 유형

**가. 기술적(Technical) 지출** — 높은 복잡성·위험, 낮은 지출
- 독과점적 공급구조, 높은 가격, 전환비용 높음
- 중점: **계약관리, 수요관리**, 경우에 따라 인소싱(In-Sourcing)

**나. 전략적(Strategic) 지출** — 높은 복잡성·위험, 높은 지출
- 복잡한 조달 프로세스, 고객화된 업무처리 필요
- 중점: **전자입찰(e-Tendering)**, 공개경쟁입찰, 기본협정(FA)
- 공공조달에서 **비중이 가장 높은** 유형

**다. 인수(Acquisition) 지출** — 낮은 복잡성·위험, 낮은 지출
- 표준화·상용화된 일상적 거래, 처리비용 절감이 중요
- 중점: **전자카탈로그(e-Catalogue)**, 전자지불, 워크플로 관리, P-card

**라. 레버리지(Leverage) 지출** — 낮은 복잡성·위험, 높은 지출
- 대규모 구매력 활용, 수요자 중심 시장
- 중점: **전자경매(e-Auction)**, 전자견적, 기본협정(FA)

### 기본협정(Framework Agreement, FA)
- 중앙조달기관(CPB) 등이 사전 검증한 공급업체와 일정기간 기본 계약조건을 미리 설정
- 입찰절차 없이 신속하게 납품요구 또는 추가 경쟁 가능
- 한국·미국의 **다수공급자계약(MAS)**과 유사

## 전자조달시스템 발전 역사

| 시기 | 발전 내용 |
|------|----------|
| 1970년대 후반 | 미국 FPDS(연방조달데이터시스템) 도입 |
| 1980~90년대 | EDI 기반 문서처리 플랫폼 사용 시작 |
| 1997년 | 한국 조달청 전자조달시스템 도입 본격화 |
| 2000년 11월 | 한국 전자입찰시스템 구축 |
| **2002년** | **KONEPS(나라장터) 구축·운용 시작** |
| 2014년 | 모든 OECD 국가 1개 이상 전자조달 플랫폼 도입 |
| 2024년 말 | 차세대 나라장터 구축 완료 |

![전자조달시스템의 발전단계 (UN, 2006)](/procurement-wiki/images/textbook/book1/p136_fig1.png)

> **학습 포인트**: UN(2006)이 제시한 전자조달시스템의 5단계 발전 모델이다. ①종이 기반 오프라인 → ②EDI/메인프레임 등 폐쇄적 전자처리 → ③이메일/웹사이트 활용 → ④통합 인터넷 기반 시스템 → ⑤전체 프로세스 통합 자동화 단계로 발전한다. KONEPS는 4~5단계에 해당하며, 시험에서 "전자조달 발전 단계"는 가끔 출제되므로 큰 흐름만 기억해두면 충분하다.

## 1.4 전자조달시스템 구성요소

전자조달은 일반적으로 통합적인 전자조달 플랫폼 형태로 구현되며, 입찰공고-입찰서 제출-평가-수주-주문-송장-지불 등 공공조달 프로세스 전 단계와 연계된 기능들로 구성된다.

![공공조달 프로세스와 전자조달시스템 기능 연계성 (UN, 2006)](/procurement-wiki/images/textbook/book1/p140_fig1.png)

> **학습 포인트**: 공공조달 프로세스 단계별로 어떤 전자조달 도구(e-Sourcing, e-Tendering, e-Auction, e-Ordering, e-Informing 등)가 매핑되는지를 보여주는 도표이다. "전자입찰=소싱~계약 단계", "전자경매=계약 단계", "전자주문=계약 체결·관리 단계"의 단계별 기능 매칭을 기억해두면 응용 문제에 강해진다.

## 핵심 암기 포인트

- 전자조달 2가지 유형: **전자적 소싱(e-Sourcing)** + **전자적 구매(e-Purchasing)**
- 지출분석 4사분면: **기술적, 전략적, 인수, 레버리지**
- 전략적 지출 = 공공조달에서 **비중 가장 높은** 유형 → e-Tendering 중심
- KONEPS 서비스 개통: **2002년**
- FA(기본협정) = 한국·미국의 **MAS(다수공급자계약)**과 유사

## OX 퀴즈

**Q1.** 전자조달시스템은 민간부문보다 공공부문의 전자적 업무처리시스템으로 주로 인식된다.
→ **O**

**Q2.** 전자적 소싱 시스템은 구매·지불 처리에 초점을 맞춘다.
→ **X** (공급자 탐색·평가에 초점. 구매·지불은 전자적 구매시스템)

**Q3.** 지출분석 4사분면 중 공공조달에서 비중이 가장 높은 유형은 전략적 지출이다.
→ **O**

**Q4.** 인수 지출 유형에서는 전자입찰(e-Tendering)이 가장 중요한 기능이다.
→ **X** (전자카탈로그, 전자지불, 워크플로 관리가 중점 기능)

**Q5.** 기본협정(FA)은 한국의 다수공급자계약(MAS)과 유사한 제도이다.
→ **O**

## 함정 노트

⚠️ **e-Sourcing vs e-Purchasing**: 소싱=공급자 탐색·평가, 구매=주문·지불 처리 — 혼동 주의

⚠️ **전략적 지출 vs 레버리지 지출**: 둘 다 높은 지출이지만, 전략적=높은 복잡성, 레버리지=낮은 복잡성

⚠️ **FA vs MAS**: 국제적으로 FA, 한국·미국에서 MAS — 사전 검증 공급업체와 기본조건 설정이라는 본질은 동일
