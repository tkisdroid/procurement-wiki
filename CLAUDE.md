# 공공조달관리사 시험 LLM Wiki

## 개요
이 프로젝트는 Karpathy의 LLM Wiki 패턴을 적용하여 공공조달관리사 시험의 모든 내용을 체계적으로 정리하는 지식 베이스입니다. LLM이 위키를 점진적으로 구축하고 유지관리합니다.

## 아키텍처

### 3계층 구조

1. **sources/** — 원본 자료 (교재, 기출문제, 법령, 판례 등). 불변의 원천 자료.
2. **wiki/** — LLM이 생성·유지하는 마크다운 파일. 요약, 개념 페이지, 법령 해설, 비교표 등.
3. **CLAUDE.md** (이 파일) — 위키 구조, 규칙, 워크플로우를 정의하는 스키마.

### 디렉토리 구조

```
procurement-wiki/
├── CLAUDE.md              # 스키마 (이 파일)
├── sources/               # 원본 자료
│   ├── textbooks/         # 교재 내용
│   ├── exams/             # 기출문제
│   ├── laws/              # 법령 원문
│   └── cases/             # 판례·사례
├── wiki/
│   ├── index.md           # 전체 위키 색인
│   ├── log.md             # 작업 로그 (append-only)
│   ├── subjects/          # 시험 과목별 개요
│   │   ├── 공공조달의이해.md
│   │   ├── 공공조달계획분석.md
│   │   ├── 공공계약관리.md
│   │   └── 공공조달관리실무.md
│   ├── concepts/          # 핵심 개념 페이지
│   ├── laws/              # 법령 해설 페이지
│   └── practice/          # 기출 분석, 출제 경향
└── web/                   # Next.js 웹 프론트엔드
```

## 위키 규칙

### 페이지 형식
모든 위키 페이지는 YAML frontmatter를 포함합니다:
```yaml
---
title: 페이지 제목
category: subjects | concepts | laws | practice
subject: 해당 과목명
parent: 상위 개념 페이지명 (있을 경우)
tags: [관련 태그]
sources: [출처 목록]
created: YYYY-MM-DD
updated: YYYY-MM-DD
importance: high | medium | low  # 교재 내 비중 기반
---
```

> **시험 특성**: 공공조달관리사는 **신규 시행 1회 시험**이므로 '빈출', '매년 출제', '자주 출제' 같은 축적된 출제 경향이 존재하지 않는다. 중요도는 교재 내 비중, 법령 근거, 실무 중요성으로 판단한다.

### 위계구조 원칙
- **세분화 우선**: 하나의 페이지가 여러 독립된 하위 개념을 포함하면 반드시 분리
- **상위 페이지**: 하위 개념의 목록과 요약, 비교표를 제공하는 허브 역할
- **하위 페이지**: 개별 개념의 상세 내용 + 시험 최적화 섹션을 포함
- **parent 필드**: 모든 하위 페이지는 frontmatter에 `parent: 상위페이지명` 기재
- **파일명 규칙**: 하위 개념은 `상위개념_하위개념.md` 형식 (예: `계약체결_수의계약.md`)
- **깊이 제한**: 최대 2단계 (과목 → 대분류 → 개별개념)

### 교차 참조
- 관련 개념은 `[[개념명]]` 형식으로 링크 (Obsidian 호환)
- 법령 참조: `[[법령명#조항]]` 형식
- 비교가 필요한 개념은 비교표로 정리

### Ingest 워크플로우
1. 새 원본 자료를 `sources/`에 추가
2. LLM이 자료를 읽고 핵심 내용 추출
3. 관련 과목 페이지 업데이트
4. 새 개념/법령 페이지 생성 또는 기존 페이지 보강
5. index.md 업데이트
6. log.md에 작업 기록 추가

## 시험 최적화 원칙

### 교재 중심 정리
- 위키의 모든 내용은 **교재**를 1차 출처로 한다
- 교재의 구성 순서와 체계를 존중하되, 시험 출제 비중에 따라 강약을 조절한다
- 교재에 없는 내용을 임의로 추가하지 않는다 (법령 원문 보충은 예외)

### 개념 페이지 필수 섹션
모든 개념 페이지는 본문 뒤에 다음 시험공부용 섹션을 포함한다:

#### 1. 핵심 암기 포인트
- 교재에서 강조하는 숫자, 기간, 요건을 bullet 형태로 정리
- 정확한 표현 사용

#### 2. OX 퀴즈
- 개념 이해를 확인할 수 있는 지문 형태로 5~10개 OX 문제 작성
- 정답과 간단한 해설 포함

#### 3. 비교 정리 (해당 시)
- 혼동하기 쉬운 유사 개념을 표로 비교

#### 4. 함정 노트
- 용어/수치 혼동이 발생하기 쉬운 포인트를 정리

### 시험 중요도 마커 (Markdown 작성 규칙)
- `> 🎯 **시험 핵심**: ...` — 교재가 강조하는 필수 암기 포인트 (빨간 박스 렌더링)
- `> 📌 **학습 포인트**: ...` — 이미지/개념 해설 (파란 박스 렌더링)
- `> ⚠️ ...` — 혼동 주의 포인트 (함정 노트, amber 렌더링)
- 본문의 `**볼드**`는 법령 용어, 수치, 핵심 명사에만 사용하고 남용하지 않는다

### Karpathy LLM Wiki 원칙
1. **출처 접지(source-grounded)**: 모든 내용은 sources/ 원본에서 추출, 임의 창작 금지
2. **점진적 구축**: 한 번에 완성하지 않고 위키를 지속 갱신
3. **교차 참조(cross-reference)**: `[[개념명]]` 링크로 지식 그래프 형성
4. **간결성(conciseness)**: 불필요한 수식어, AI 티 나는 장황한 표현 배제
5. **계층화(hierarchical)**: 허브 페이지 + 세부 페이지 구조

### 중요한 점
내용이 틀린점이 있어서는 절대 안되므로, 내용이 생성될 때마다, 자체검증을 실시하고, 틀릴경우 web search를 이용해서 내용을 검증합니다.
